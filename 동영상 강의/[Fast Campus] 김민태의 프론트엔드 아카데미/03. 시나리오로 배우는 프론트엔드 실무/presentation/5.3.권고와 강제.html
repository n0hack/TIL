(표준) 권고와 강제

권고는 작동하는가?

eg. Airbnb Javascript 가이드를 준수해주세요.

권고가 작동하지 않게되는 순서

  • 업무는 언제나 우선순위가 작동한다
  • 0순위 우선순위 발생의 예외가 늘 일어난다
  • 2순위는 처리되지 않고 방치된다
  • 권고는 2순위로 배정될 가능성이 높다

강제가 정답

강제가 발생시키는 문제점을 인지하고 해소 활동이 병행되어야 한다

강제의 문제점

  • 개성이 말살된다는 심리적 불만 누적
  • 변화에 둔감한 보수적 조직이라는 평가

강제의 문제점 - 해결책

5.2 개성을 다루는 방법

표준의 명분과 대상

명분

제품 라이프사이클과 개발자 라이프사이클의 차이

대상

  • 작업 단위, 플래닝 단위 가이드
  • 코딩 컨벤션
  • 어플리케이션 아키텍처(빌드, 배포, 레포 구조 등)
  • 예외 처리 아키텍처
  • 성능 가이드라인 (최소 품질 가이드라인)
  • 메세지 처리 방식
  • 리뷰, 머지, 릴리즈 규칙
